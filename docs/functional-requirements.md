## Overview

A Python script that generates a 6-day work plan and stream schedule by analyzing:
- Meta-roadmap strategic priorities
- Project health data (via `portfolio_status.py`)
- GitHub milestones and open issues
- Task time estimates and streamability

### Key Features

1. **Automated Weekly Planning**
   - Work plan: 6 hours/day (4 streaming + 2 off-stream)
   - Stream schedule: Mon-Fri, 1-5 PM CST, grouped by project
   - **Runs Sunday night ‚Üí entire week planned**

2. **Smart Task Allocation**
   - Prioritizes based on pillar, urgency, and importance
   - Groups by project to minimize context switching
   - Respects time estimates from issue descriptions
   - Prefers grouping tasks by milestone for cohesive streams

3. **Automated Stream Plan Generation**
   - **Generates 5 complete stream folders** (one per day)
   - Each folder contains: outline, checklist, social posts, YouTube description
   - Output to `Streaming/streams/` ready to use
   - Eliminates manual `/stream-plan` invocation before each stream

4. **Multiple Output Formats**
   - Markdown work plan (human-readable)
   - ICS calendar file (importable to calendar apps)
   - 5 stream plan folders (all promo content pre-written)

## Core Logic Flow

### 1. Data Gathering

**Sources:**
- Reuse `lib/github_client.py` from `portfolio_status.py`
- Fetch all open issues across portfolio projects
- Parse `meta-roadmap.md` for current quarter priorities
- Load project configuration from `config.yaml`

**Issue Data Needed:**
- Title, number, repository
- Labels (especially `streamable`)
- Time estimate (parsed from description)
- Milestone association
- Priority labels
- Assignee

### 2. Time Estimation

**Parse patterns from issue descriptions:**

```python
# Regex patterns to match:
estimate_patterns = [
    r"(?i)estimate[d]?:\s*(\d+)\s*(?:hours?|hrs?|h)",  # "Estimate: 2 hours"
    r"(?i)time:\s*(\d+)\s*(?:hours?|hrs?|h)",          # "Time: 3h"
    r"\[(\d+)h\]",                                      # "[2h]"
]

# Default: 1 hour if no estimate found
```

**Examples:**
- "Estimate: 2 hours" ‚Üí 2h
- "This will take 3h to complete" ‚Üí 3h
- "[4h] Build the feature" ‚Üí 4h
- No estimate found ‚Üí 1h (default)

### 3. Task Classification

**Streamable vs Off-Stream:**

```python
def is_streamable(issue):
    """
    Determine if a task can be streamed.

    Rules:
    - Has 'streamable' label ‚Üí YES
    - No 'streamable' label ‚Üí NO (off-stream only)
    """
    return 'streamable' in [label.lower() for label in issue.labels]
```

**Task Types:**
- **Streamable**: Feature development, bug fixes, refactoring, UI work
- **Off-stream**: Planning, research, PR reviews, documentation, meetings

### 4. Prioritization Algorithm

**Score Calculation:**

```python
score = (pillar_weight √ó urgency √ó importance) / time_hours
```

**Pillar Weights** (from meta-roadmap):
- Revenue (EmberCare): `10`
- Infrastructure (RAPID, github-org-toolkit): `7`
- Consistency (Streaming): `5`
- Innovation (RapidOS): `3`
- Cleanup (acreetion_os-Org): `1`

**Urgency Multipliers:**
- Blocking other work: `10`
- Due this week: `8`
- Due next 2 weeks: `5`
- Due this month: `4`
- No deadline: `3`

**Importance Multipliers:**
- Critical path item: `10`
- Decision gate preparation: `8`
- Milestone completion: `6`
- Normal priority: `4`
- Low priority: `2`

**Example:**
```
Issue: EmberCare #123 - "Complete medication form validation [2h]"
- Pillar: Revenue (10)
- Urgency: Due this week (8)
- Importance: Critical path (10)
- Time: 2 hours

Score = (10 √ó 8 √ó 10) / 2 = 400
```

### 5. Time Packing Algorithm

**Daily Schedule Template:**
```
Mon-Fri:
  Stream Block:   1:00 PM - 5:00 PM CST (4 hours)
  Off-Stream:     Flexible (2 hours, mornings/evenings)
```

**Packing Rules:**

1. **Stream Block (4 hours)**
   - Only streamable tasks
   - Prefer single project per day (minimize context switching)
   - Pack tasks until 4 hours filled or no more streamable tasks
   - Sort by priority score (highest first)

2. **Off-Stream Block (2 hours)**
   - Non-streamable tasks OR overflow streamable tasks
   - Can be different project than stream
   - Planning, reviews, research, testing

**Algorithm:**

```python
for day in week_days:
    stream_tasks = []
    stream_hours = 0

    # Pack streamable tasks (4-hour target)
    for task in sorted_streamable_tasks:
        if stream_hours + task.hours <= 4:
            stream_tasks.append(task)
            stream_hours += task.hours
        if stream_hours >= 4:
            break

    # Pack off-stream tasks (2-hour target)
    offstream_tasks = []
    offstream_hours = 0

    for task in sorted_offstream_tasks:
        if offstream_hours + task.hours <= 2:
            offstream_tasks.append(task)
            offstream_hours += task.hours
        if offstream_hours >= 2:
            break

    schedule[day] = {
        'stream': stream_tasks,
        'offstream': offstream_tasks
    }
```

**Context Switching Optimization:**

```python
# Group streamable tasks by project
tasks_by_project = group_by(streamable_tasks, key='repository')

# For each day, try to assign a single project
for day in week_days:
    # Pick highest priority project with enough tasks to fill 4 hours
    best_project = find_best_fit_project(tasks_by_project, target_hours=4)
    assign_to_day(day, best_project)
```

### 6. Milestone Grouping Strategy

**Goal:** Create cohesive streams by grouping tasks from the same milestone when possible.

**Hybrid Approach:**

1. **Prefer Milestone Grouping**
   - If a milestone has 3-5 hours of streamable tasks ‚Üí dedicate a stream day to it
   - More cohesive stream narrative
   - Better viewer experience (logical progression)

2. **Fall Back to Priority Packing**
   - If no milestones meet the threshold ‚Üí pack by priority score
   - Mix tasks from different milestones if needed to fill 4 hours
   - Ensure high-priority work still gets done

**Algorithm:**

```python
def allocate_streaming_days(streamable_tasks: list, num_days: int = 5) -> dict:
    """
    Allocate streamable tasks to streaming days.

    Priority: Milestone cohesion > Priority score
    """
    schedule = {}
    remaining_tasks = streamable_tasks.copy()

    # Group by milestone
    milestone_groups = group_tasks_by_milestone(remaining_tasks)

    # Phase 1: Assign milestone-cohesive days
    for milestone, tasks in sorted(
        milestone_groups.items(),
        key=lambda x: sum(t['priority_score'] for t in x[1]),
        reverse=True
    ):
        total_hours = sum(t['hours'] for t in tasks)

        if 3 <= total_hours <= 5:  # Good fit for 4-hour stream
            day = next_available_day(schedule)
            schedule[day] = {
                'tasks': tasks[:4],  # Limit to fill ~4 hours
                'milestone': milestone,
                'cohesive': True
            }
            remaining_tasks = [t for t in remaining_tasks if t not in tasks]

    # Phase 2: Fill remaining days with priority-based packing
    for day in range(1, num_days + 1):
        if day not in schedule:
            packed_tasks = pack_by_priority(remaining_tasks, target_hours=4)
            schedule[day] = {
                'tasks': packed_tasks,
                'milestone': 'Mixed',
                'cohesive': False
            }
            remaining_tasks = [t for t in remaining_tasks if t not in packed_tasks]

    return schedule
```

**Benefits:**
- More engaging streams (coherent story arc)
- Easier to generate stream titles and descriptions
- Matches `/stream-plan` workflow (which requires milestone selection)
- Better YouTube algorithm performance (clear topic focus)

**Edge Cases:**
- **No milestones defined:** Fall back to priority packing entirely
- **All milestones too small (<3h):** Combine related milestones or pack by project
- **All milestones too large (>5h):** Split milestone across multiple days

### Advanced Options

```bash
# Custom number of days
python scripts/weekly_planner.py --days 3

# Specify start date
python scripts/weekly_planner.py --start-date 2025-11-04

# Custom output directory
python scripts/weekly_planner.py --output /path/to/output/

# Dry run (print to terminal, don't write files)
python scripts/weekly_planner.py --dry-run

# Verbose mode (show prioritization scores)
python scripts/weekly_planner.py --verbose

# Skip calendar generation
python scripts/weekly_planner.py --no-calendar

# Generate work plan only (skip stream plan generation)
python scripts/weekly_planner.py --no-stream-plans

# Regenerate stream plan for specific date
python scripts/weekly_planner.py --regenerate-stream-plan 2025-11-04

# Force regenerate all stream plans (overwrites existing)
python scripts/weekly_planner.py --force-regenerate-streams
```

### Help

```bash
python scripts/weekly_planner.py --help
```
